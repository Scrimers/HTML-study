CSS란

Cascading Style Sheets의 줄임말.
웹페이지를 꾸미는 디자인 요소로서 쓰인다.
HTML이 만들어진 후 사람들은 단순히 정보를 노출 시키는 것에 불만을 표했다.
이를 보완하기 위해 디자인 적 요소가 추가되게 되는데
먼저 생긴 것이 font 태그와 같이 기존 html에 디자인과 관련된 태그를 추가하는 것이었다.
하지만 수많은 페이지의 수많은 부분의 디자인을 변경해야할 경우 font 태그나 inline-style만으로는 유지보수가 힘들어진다는 한계점에 도달하게 된다.
그리하여 HTML에서 스타일만 분리하여 반복적인 부분을 일괄적으로 적용하기 위해 CSS가 탄생하게 되었다.

CSS는 HTML과는 완전히 다른 언어이지만 서로 호환이 된다.
이유는 CSS가 HTML문법으로 작성되었기에 이를 웹브라우저가 인식하여 CSS 문법으로 해석할 수 있기에 호환이 가능한 것이다.


내부 스타일(internal style)

head 태그 밑에 style태그와 선택자를 이용하여 적용할 CSS를 적는 방법

i태그만 색상 빨간색으로 바꿈
style태그 = 웹브라우저가 CSS를 인식하기 위한 태그
i = 이탤릭태그를 지닌 모든 것들 (선택자 = selector)
color:red = 효과
color = 속성
red = 값
; = 효과의 끝부분을 지정

내부 스타일의 장점 : 선언한 선택자를 지닌 모든 것들에 일괄적으로 CSS를 적용 가능. 즉, 유지보수 용이
내부 스타일의 단점 : 같은 선택자를 지녔으나 서로 다른 CSS를 적용해야 하는 경우 불가능함.
(이를 해결하기 위해선 선택자가 상세하게 나뉘어야 한다.)

//<head>아래에 <style> i { color:red; } </style>

인라인 스타일(inline-style)에 대하여

HTML태그의 속성란에 style을 선언하여 적용할 CSS를 적는 방법
같은 a 태그지만

이건 윗줄이 있다
이건 줄이 없다
이건 밑줄이 있다

인라인 스타일의 장점 : 각 요소마다 개별적으로 CSS를 적용시킬 수 있다.
인라인 스타일의 단점 : 중복되는 여러 요소들에도 전부 개별적으로 CSS를 적용시켜야 하니 유지보수가 어려워짐.

<a href="" style="text-decoration: overline;">이건 윗줄이 있다</a> <a href="" style="text-decoration: none;">이건 줄이 없다</a> <a href="">이건 밑줄이 있다</a>

태그를 선택할 때는 아래와 같다. <a href="">a태그</a> 라는 html 문구가 있다고 쳤을떄 a { color : red; }
클래스를 선택할 떄는 아래와 같다.
주의할 점은 일반 태그와는 달리 클래스라는 것을 구분하기 위해 .이 클래스명 앞에 쓰여야 한다는 것이다. <a href="" class="aTag">a태그</a> 라는 html 문구가 있다고 쳤을떄 .aTag { color : red; }
아이디를 선택할 떄는 아래와 같다.
주의할 점은 일반 태그와는 달리 아이디라는 것을 구분하기 위해 #이 아이디 앞에 쓰여야 한다는 것이다. <a href="" class="aTag" id="aTagId">a태그</a> 라는 html 문구가 있다고 쳤을떄 #aTagId { color : red; }

결합자에 대하여

서로 다른 여러 태그, 클래스, 아이디를 한번에 선택할 수도 있다.
선택자 목록은 다음과 같다
아래와 같은 html 문구가 있다. <a href="" class="aClass" id="aId">a 태그</a> <p>p 태그</p> <p>부모 p 태그 <a href="" class="bClass">자식 a 태그</a> </p> 그리고 그에 대한 CSS가 아래와 같이 있을때 a, p { color : yellowgreen; } 모든 a, p태그는 전부 yellowgreen 색상으로 변하게 된다.
자손 결합자는 다음과 같다.
아래와 같은 html 문구가 있다. <p>p 부모 <a href="">a자식</a> <button>button자손</button> <p>p 부모2 <button>button자손</button> </p> <p>p 부모3 <button>button자손</button> </p> </p> 그리고 그에 대한 CSS가 아래와 같이 있을때 p, button { color : yellowgreen; } 제일 처음의 p태그 아래에 있는 모든 button태그는 전부 yellowgreen 색상으로 변하게 된다.
자식 결합자는 다음과 같다.
아래와 같은 html 문구가 있다. <p>p 부모 <a href="">a자식</a> <h6>p 부모2 <button>button자손</button> </h6> <h6>p 부모3 <button>button자손</button> </h6> </p> 그리고 그에 대한 CSS가 아래와 같이 있을때 p > a { color : red; } p > button { color : blue; } 제일 처음의 p태그 바로 아래에 있는 a태그만 red로 변하게 된다.
button의 색상이 변하지 않는 이유는 자식이 아닌 자손이기 떄문이다.

CSS 적용 순서에 대하여

CSS의 적용순서는 선택자에 따라 우선순위가 결정되고
같은 우선순위의 선택자인 경우 가장 마지막에 선언된것이 적용된다.
아래와 같은 html 문구가 있을때 <a href="" class="aColor bColor" id="aTagId">a태그</a> 그리고 그에 대한 CSS가 아래와 같이 있을때 .bColor { color : rgb(225, 255, 0); font-size: 50px; } #aTagId { color : rgb(19, 166, 44); } a { color : black; } .aColor { color : rgb(0, 229, 255); } 적용되는 CSS는 #aTagId의 color: rgb(225, 255, 0);와 .bColor의 font-size: 50px; 이다.
아이디 선택자가 우선적으로 적용되므로 color 속성을 지닌 모든 것들 중에서 아이디 선택자의 것이 적용되는 것이고
유일하게 font-size 속성을 지닌 bColor선택자의 속성이 적용되는 것이다.
만약 같은 우선순위의 같은 속성인 경우 아래와 같이 된다.
아래와 같은 html 문구가 있을때 <a href="" class="aColor bColor cColor">a태그</a> 그리고 그에 대한 CSS가 아래와 같이 있을떄 .cColor { color : black; font-size: 10px; } .bColor { color : red; font-size: 50px; } .aColor { color : yellow; } 선택자는 같은 클래스이므로 가장 마지막에 선언된 .aColor가 우선순위를 갖게 되어 color : yellow; 가 적용되게 된다.
또한 .aColor와 중복되지 않는 font-size의 경우 .bColor가 .cColor보다 아래에 위치하기 떄문에 font-size: 50px; 이 적용되게 된다.

중복을 줄이는 방법에 대하여

아래와 같은 HTML이 있고 <h1>h1태그</h1> 아래와 같은 CSS가 있을때 h1 { border-width: 3px; border-color: red; border-style: solid; } 위와 같은 CSS문구를 아래와 같이 줄여서 쓸 수 있다. h1 { border: solid red 5px } 이때 순서는 중요치 않다. red solid 5px 라고 해도 된다.

display에 대하여

display : inline = 컨텐츠 너비만큼 영역이 잡힘
<h1>h1태그</h1> h1 { background-color :red; display: inline; }
display : block = 화면 전체 너비만큼 영역이 잡힙
<h1>h1태그</h1> h1 { background-color :red; display: block; } ※둘 다 높이는 컨텐츠의 높이를 따라간다.

박스 모델에 대하여

border = 테두리
margin = 테두리 기준 외부 여백
padding = 테두리 기준 내부 여백
<h2>h2태그</h2> <h2 class="in">h2태그</h2> <h1>h1태그</h1> <h2 class="in">h2태그</h2> <h2>h2태그</h2> h2 { background-color: red; padding: 30px; } .in { display: inline; } h1 { display: inline; border: 2px blue solid; margin: 10px; padding: 10px; }

그리드에 대하여

div : division의 줄임말. 단순히 디자인의 용도로만 쓰임.
기본적으로 block level element 이다.
span : 단순히 디자인의 용도로만 쓰임.
기본적으로 inline level element 이다.
div들을 나란히 배치하고싶다면 부모div가 필요하다.
아래와 같이 하면 된다.
※fr = 화면전체를 쓸 수 있도록 자동으로 조정되는 단위 <div id="grid"> <div>navi</div> <div>art</div> </div> #grid { border: 5px solid pink; display: grid; /* grid-template-columns: 150px 1fr; //navi는 무조건 150px, art는 남은 공간을 차지 grid-template-columns: 1fr 1fr; //navi와 art가 같은 비율로 전체 공간을 차지 grid-template-columns: 2fr 1fr; //전체 화면을 3등분으로 봤을때 navi는 2, art는 1을 차지 */ } div{ border: 5px solid gray; }

반응형 디자인에 대하여

반응형 디자인 : 화면의 크기에 따라서 웹페이지의 각 요소들이 최적화된 모양으로 바뀌게 하는 것.
이러한 반응형 디자인을 만들기 위해선 미디어쿼리(@media)를 써야한다. <div>Responsive</div> div { border: 10px solid green; font-size: 60px; } @media(max-width:600px) { div { display: none; } } 위의 내용은 div태그는 기본적으로 border, font-size속성을 지니고 있되
화면의 최대 크기가 600이하일떄는 div태그를 보이지 않게 하라는 의미이다.
